# 서브에이전트 패턴

## 오늘의 목표

> 하나의 Claude가 다른 Claude를 부리는 구조 이해하기

지금까지 Claude 하나가 혼자 일하는 법을 배웠습니다. 하지만 복잡한 프로젝트에서는 **한 명이 모든 걸 하는 것보다 팀으로 움직이는 게 효율적**입니다. 서브에이전트는 Claude에게 팀을 만들어주는 개념입니다.

---

## 서브에이전트란?

회사 조직을 떠올려보세요. 팀장이 모든 일을 혼자 하지 않습니다. “A씨는 디자인, B씨는 백엔드, C씨는 테스트”처럼 **일을 나눠서 맡깁니다**. 각 팀원은 자기 분야를 끝내고 결과를 팀장에게 보고합니다.

서브에이전트도 같습니다. **메인 Claude(팀장)**가 작업을 분석하고, 부분 작업을 **서브에이전트 Claude(팀원)**에게 나눠줍니다.

`┌──────────────────────────┐
│    메인 에이전트 (팀장)     │
│    "전체 리팩토링해줘"      │
└─────────┬────────────────┘
          │
    ┌─────┼─────┐
    │     │     │
    ▼     ▼     ▼
 ┌─────┐ ┌─────┐ ┌─────┐
 │서브1 │ │서브2 │ │서브3 │
 │API  │ │UI   │ │테스트│
 │리팩 │ │리팩 │ │작성 │
 └─────┘ └─────┘ └─────┘`
---

## Claude Code의 Agent 도구

Claude Code에는 **Agent**라는 내장 도구가 있습니다. 이 도구가 바로 서브에이전트를 생성합니다.

메인 Claude가 작업을 수행하다가 “이 부분은 따로 맡기는 게 낫겠다”고 판단하면 Agent 도구를 사용해서 서브에이전트를 만듭니다. 서브에이전트는 독립된 컨텍스트에서 작업하고, 결과만 돌려줍니다.

### 왜 나눠서 처리할까?

| 장점 | 설명 |
| --- | --- |
| **컨텍스트 분리** | 각 서브에이전트가 자기 작업에만 집중합니다 |
| **병렬 처리** | 여러 작업을 동시에 진행할 수 있습니다 |
| **안정성** | 서브에이전트 하나가 실패해도 다른 작업에 영향 없습니다 |
| **효율성** | 큰 작업을 작은 조각으로 나누면 각각의 품질이 올라갑니다 |

> ℹ️ **정보**
>
> 사람이 직접 서브에이전트를 만들 필요는 없습니다. Claude Code가 필요하다고 판단하면 **자동으로** Agent 도구를 사용합니다. 우리는 이 구조를 이해하고 있으면 됩니다.

## 병렬 실행: 여러 서브에이전트 동시에

서브에이전트의 가장 큰 장점은 **동시 실행**입니다.

예를 들어 “이 프로젝트의 모든 컴포넌트를 TypeScript로 변환해줘”라고 요청하면, Claude는 이렇게 할 수 있습니다.

`메인 에이전트:
  "10개 컴포넌트를 변환해야 하니, 나눠서 처리하겠습니다"

  서브에이전트 1 → Header.jsx → Header.tsx
  서브에이전트 2 → Sidebar.jsx → Sidebar.tsx
  서브에이전트 3 → Footer.jsx → Footer.tsx
  ...`
순서대로 하면 30분 걸릴 작업이, 동시에 처리하면 훨씬 빠릅니다. 마치 팀원 여러 명이 동시에 작업하는 것과 같습니다.

## 백그라운드 실행

Agent SDK와 결합하면 서브에이전트를 **백그라운드에서 실행**할 수도 있습니다. 메인 에이전트가 다른 작업을 하는 동안 서브에이전트가 뒤에서 묵묵히 일합니다.

`import { claude } from "@anthropic-ai/claude-code";
 
async function parallelWork() {
  // 3개 작업을 동시에 실행
  const [review, test, docs] = await Promise.all([
    claude({
      prompt: "src/api/ 폴더의 코드를 리뷰해줘",
      options: { maxTurns: 5, allowedTools: ["Read", "Grep", "Glob"] }
    }),
    claude({
      prompt: "src/utils/ 폴더의 테스트 커버리지를 분석해줘",
      options: { maxTurns: 5, allowedTools: ["Read", "Grep", "Glob", "Bash"] }
    }),
    claude({
      prompt: "README.md가 코드와 일치하는지 확인해줘",
      options: { maxTurns: 3, allowedTools: ["Read", "Grep", "Glob"] }
    }),
  ]);
 
  console.log("리뷰:", review.stdout);
  console.log("테스트:", test.stdout);
  console.log("문서:", docs.stdout);
}`
`Promise.all`로 3개의 Claude를 동시에 실행합니다. 각각 독립적으로 작업하고 모두 끝나면 결과를 모아줍니다.

## 워크트리 격리

서브에이전트가 파일을 수정하는 경우, 서로의 작업이 충돌할 수 있습니다. 같은 파일을 동시에 수정하면 문제가 생기겠죠.

이를 방지하는 것이 **워크트리 격리**입니다. Git의 worktree 기능을 활용해서, 각 서브에이전트가 **별도의 작업 공간**에서 일합니다.

`메인 프로젝트/
├── .git/
├── src/                      ← 원본 (건드리지 않음)
│
├── .worktrees/
│   ├── agent-1/src/          ← 서브에이전트 1의 작업 공간
│   ├── agent-2/src/          ← 서브에이전트 2의 작업 공간
│   └── agent-3/src/          ← 서브에이전트 3의 작업 공간`
비유하자면, 같은 문서를 수정할 때 **각자 복사본을 만들어서 작업하고**, 다 끝나면 합치는 것입니다. 원본은 안전합니다.

> ℹ️ **정보**
>
> 워크트리 격리는 고급 패턴입니다. 지금은 “서브에이전트가 서로 충돌하지 않게 하는 방법이 있다”는 것만 기억해두세요.

## AI싱크클럽 실전 패턴

우리 AI싱크클럽의 youtube_maker 프로젝트가 바로 서브에이전트 패턴의 실전 예시입니다.

`youtube-orchestrator (팀장)
├── trend-collector (트렌드 수집 팀원)
├── keyword-researcher (키워드 조사 팀원)
├── competitor-analyzer (경쟁 분석 팀원)
├── script-planner (기획 팀원)
├── hook-generator (훅 제작 팀원)
├── script-writer (대본 작성 팀원)
├── retention-optimizer (리텐션 최적화 팀원)
├── script-reviewer (리뷰 팀원)
└── ... (총 21개 에이전트)`
오케스트레이터(팀장)가 “이번 영상 만들어줘”라는 요청을 받으면, 각 단계에 맞는 전문 에이전트(팀원)를 순서대로 호출합니다. Phase 0에서는 3개 에이전트가 **병렬로** 소재를 수집하고, 그 결과를 모아서 다음 Phase로 넘깁니다.

### 패턴의 핵심

| 원칙 | 적용 |
| --- | --- |
| **전문 분업** | 각 에이전트가 하나의 역할만 담당 |
| **순차 + 병렬 혼합** | Phase 내부는 병렬, Phase 간에는 순차 |
| **컨텍스트 전달** | 이전 Phase의 산출물을 다음 Phase에 전달 |
| **독립 실행 가능** | 각 에이전트를 단독으로도 실행 가능 |

> ℹ️ **정보**
>
> 21개 에이전트라니 복잡해 보이지만, 각각은 단순한 Skills 파일입니다. 복잡한 시스템도 결국 **단순한 조각의 조합**입니다. Day 6에서 배운 Skills, Headless, Agent SDK가 모두 여기에 쓰입니다.

## 서브에이전트를 설계할 때 체크리스트

직접 서브에이전트 시스템을 만들고 싶다면, 이 체크리스트를 참고하세요.

1. **작업을 독립 단위로 나눌 수 있는가?** — 의존성이 적을수록 좋습니다

1. **각 에이전트의 입력/출력이 명확한가?** — “무엇을 받고 무엇을 내놓는지” 정의합니다

1. **실패 시 복구가 가능한가?** — 하나가 실패해도 전체가 멈추지 않게 합니다

1. **병렬 실행이 가능한 부분이 있는가?** — 있다면 `Promise.all`로 동시 실행합니다

1. **결과를 합치는 로직이 있는가?** — 오케스트레이터가 전체를 조율합니다

## 정리

- 서브에이전트는 **메인 Claude가 부분 작업을 다른 Claude에게 위임**하는 패턴입니다

- Claude Code의 Agent 도구가 자동으로 서브에이전트를 생성합니다

- **병렬 실행**으로 작업 속도를 높일 수 있습니다

- **워크트리 격리**로 서브에이전트 간 충돌을 방지합니다

- AI싱크클럽은 21개 전문 에이전트가 협업하는 실전 시스템을 운영하고 있습니다
