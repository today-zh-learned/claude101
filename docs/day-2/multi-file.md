# 여러 파일 한번에 다루기

## 오늘의 목표

> Claude Code로 프로젝트 전체를 파악하고, 여러 파일을 한번에 다루는 법을 배웁니다.

지금까지는 파일 하나를 읽고, 하나를 수정하는 작업을 했습니다. 하지만 실제 프로젝트는 파일이 수십, 수백 개입니다. Claude Code의 진짜 힘은 **프로젝트 전체를 이해하고 여러 파일을 동시에 다루는 것**에 있습니다.

---

## 단일 파일 vs 멀티파일 작업

채팅 AI(ChatGPT, Claude.ai)에서는 파일을 하나씩 업로드하고, 하나씩 결과를 받습니다. 10개 파일을 수정하려면 10번 복사-붙여넣기를 해야 합니다.

Claude Code는 다릅니다.

|  | 채팅 AI | Claude Code |
| --- | --- | --- |
| 파일 분석 | 하나씩 업로드 | 프로젝트 전체를 한눈에 |
| 파일 수정 | 결과를 복사해서 붙여넣기 | 직접 여러 파일을 동시에 수정 |
| 맥락 파악 | 파일 간 연결을 모름 | 파일 간 관계를 이해 |
| 예시 | ”이 파일 고쳐줘" | "이 기능과 관련된 파일 전부 고쳐줘” |

이 차이가 Claude Code를 쓰는 가장 큰 이유입니다.

---

## ”이 프로젝트의 구조 설명해줘”

프로젝트를 처음 열었을 때, 이렇게 물어보세요.

`> 이 프로젝트의 구조를 설명해줘`
Claude가 폴더와 파일을 탐색해서 전체 구조를 정리해줍니다.

`이 프로젝트는 React 기반의 웹 애플리케이션입니다.

src/
  components/  — UI 컴포넌트 (Button, Header 등)
  pages/       — 페이지별 라우트
  hooks/       — 커스텀 훅
  utils/       — 유틸리티 함수
  api/         — API 호출 함수
public/        — 정적 파일 (이미지 등)
package.json   — 의존성 목록`
다른 사람이 만든 프로젝트를 분석할 때 특히 유용합니다. 코드를 한 줄도 읽지 않아도 전체 그림을 파악할 수 있습니다.

> ℹ️ **정보**
>
> 프로젝트를 파악하는 다른 질문들도 시도해보세요. “이 프로젝트에서 가장 중요한 파일 5개가 뭐야?”, “엔트리 포인트가 어디야?”, “이 프로젝트 어떻게 실행해?” 같은 질문도 잘 답합니다.

## 여러 파일에 걸친 변경 요청하기

Claude Code의 진가가 발휘되는 순간입니다. 한 번의 요청으로 여러 파일을 동시에 수정할 수 있습니다.

### 예시 1: 새 기능 추가

`> 로그인 페이지를 만들어줘. 컴포넌트, 페이지, API 호출 함수까지 전부`
Claude가 한 번에 처리합니다:

- `src/components/LoginForm.tsx` — 로그인 폼 컴포넌트 생성

- `src/pages/LoginPage.tsx` — 로그인 페이지 생성

- `src/api/auth.ts` — 인증 API 함수 생성

- `src/App.tsx` — 라우트에 로그인 페이지 추가

4개 파일을 만들거나 수정하는데, 여러분은 한 번만 요청했습니다.

### 예시 2: 스타일 일괄 변경

`> 모든 버튼 컴포넌트의 기본 색상을 파란색에서 초록색으로 바꿔줘`
Claude가 버튼 관련 파일을 모두 찾아서 한꺼번에 수정합니다.

### 예시 3: 에러 추적

`> "Cannot read property of undefined" 에러가 나는데, 원인 찾아서 고쳐줘`
Claude가 관련 파일을 추적하면서 에러의 원인을 찾고, 필요한 파일을 수정합니다.

## 리팩토링 실습: “변수명 전체 바꿔줘”

리팩토링은 **코드의 동작은 그대로 두고, 구조나 이름을 개선하는 작업**입니다. 여러 파일에 걸쳐 있어서 수동으로 하면 실수하기 쉽습니다.

**Step 1**: Claude Code를 프로젝트 폴더에서 실행합니다.

`cd ~/my-project
claude`
**Step 2**: 변수명 변경을 요청합니다.

`> "userData"라는 변수명을 프로젝트 전체에서 "userProfile"로 바꿔줘`
**Step 3**: Claude가 해당 변수가 사용된 모든 파일을 찾아서 바꿔줍니다.

`변경된 파일:
- src/hooks/useUser.ts (3곳)
- src/pages/ProfilePage.tsx (2곳)
- src/api/user.ts (1곳)
- src/components/UserCard.tsx (2곳)
총 4개 파일, 8곳 변경`
수동으로 하면 하나를 빠뜨리거나 엉뚱한 곳을 바꿀 수 있습니다. Claude는 **맥락을 이해하기 때문에** 변수명만 정확히 바꿉니다. 문자열 안의 “userData”나 주석의 “userData”는 상황에 맞게 처리합니다.

## 파일 검색하는 법

Claude Code는 프로젝트에서 파일을 찾을 때 내부적으로 전문 도구를 사용합니다.

### Glob: 파일 이름으로 찾기

`> 이 프로젝트에서 .tsx 파일이 몇 개인지 알려줘`
Claude가 `Glob` 도구를 사용해서 `**/*.tsx` 패턴으로 파일을 찾습니다.

### Grep: 파일 내용으로 찾기

`> "TODO"라고 적힌 곳 전부 찾아줘`
Claude가 `Grep` 도구를 사용해서 프로젝트 전체에서 “TODO”가 포함된 줄을 찾아줍니다.

여러분은 Glob이나 Grep을 직접 쓸 필요가 없습니다. **자연어로 물어보면** Claude가 알아서 적절한 도구를 선택합니다.

`> 이 프로젝트에서 axios를 쓰는 파일 보여줘
> 테스트 파일들 어디에 있어?
> console.log가 남아있는 곳 찾아줘`
모두 Claude가 검색 도구를 활용해서 답해줍니다.

> ℹ️ **정보**
>
> 큰 프로젝트일수록 이 기능이 빛납니다. 파일이 200개인 프로젝트에서 특정 함수가 어디서 쓰이는지 찾는 건, 사람이 하면 30분이지만 Claude에게 시키면 10초입니다.

## 실전 패턴: 이렇게 시켜보세요

멀티파일 작업에서 유용한 요청 패턴을 모았습니다.

| 상황 | 이렇게 말하세요 |
| --- | --- |
| 프로젝트 파악 | ”이 프로젝트 구조 설명해줘” |
| 기능 추가 | ”회원가입 기능 만들어줘. 필요한 파일 전부” |
| 일괄 수정 | ”모든 API 함수에 에러 처리 추가해줘” |
| 변수명 변경 | ”fetchData를 loadUserData로 전부 바꿔줘” |
| 코드 찾기 | ”결제 관련 코드가 어디에 있어?” |
| 의존성 확인 | ”이 컴포넌트를 삭제하면 어디가 깨져?” |
| 정리 | ”안 쓰는 import 전부 정리해줘” |

핵심은 **구체적으로** 요청하는 것입니다. “코드 좀 고쳐줘”보다 “모든 API 함수에 try-catch 추가해줘”가 훨씬 좋은 결과를 냅니다.

## 주의할 점

멀티파일 작업이 강력한 만큼, 주의할 점도 있습니다.

1. **커밋 후 작업하세요**: 큰 변경을 시키기 전에 현재 상태를 커밋해두면, 마음에 안 들 때 되돌릴 수 있습니다

1. **한 번에 너무 많이 시키지 마세요**: “프로젝트 전체를 리팩토링해줘”보다 “이 폴더의 파일들을 리팩토링해줘”가 결과가 좋습니다

1. **결과를 확인하세요**: 특히 처음에는 Claude가 수정한 파일을 하나씩 확인하는 습관이 좋습니다

> ⚠️ **주의**
>
> Claude Code가 수정한 파일이 많을 때, “뭘 바꿨는지 요약해줘”라고 물어보면 변경 내역을 정리해줍니다. 전부 직접 확인하지 않아도 빠르게 파악할 수 있습니다.

## 정리

- Claude Code는 **프로젝트 전체를 이해**하고 여러 파일을 동시에 다룹니다

- “프로젝트 구조 설명해줘”로 전체 그림을 파악할 수 있습니다

- 한 번의 요청으로 **여러 파일을 생성하거나 수정**할 수 있습니다

- 리팩토링, 변수명 변경 같은 일괄 작업이 특히 강력합니다

- Claude가 내부적으로 Glob, Grep 같은 검색 도구를 사용하므로, **자연어로 물어보면** 알아서 찾아줍니다
