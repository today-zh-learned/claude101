# 권한 설정 이해하기

## 오늘의 목표

> Claude가 물어보는 권한이 뭔지 이해하고, 편하게 설정합니다.

Claude Code를 쓰다 보면 “이 작업을 실행해도 될까요?”라고 물어보는 순간이 있습니다. 이게 뭔지, 어떻게 설정하면 편한지 알아봅시다.

---

## Claude는 왜 물어볼까요?

Claude Code는 여러분의 컴퓨터에서 **직접** 파일을 만들고, 수정하고, 명령어를 실행합니다. 강력한 만큼, 실수하면 파일이 날아가거나 의도하지 않은 변경이 생길 수 있습니다.

그래서 Claude Code는 **위험할 수 있는 작업**을 하기 전에 허락을 구합니다.

`Claude wants to execute: rm -rf dist/
Allow? (y/n)`
이런 메시지가 뜨면:

- **y를 누르면**: 작업을 진행합니다

- **n을 누르면**: 작업을 건너뜁니다

매번 물어보는 게 안전하지만, 작업 흐름이 끊기기도 합니다. 그래서 Claude Code에는 **세 가지 권한 모드**가 있습니다.

---

## 세 가지 권한 모드

### 1. 신중한 모드 (기본값)

파일을 수정하거나 명령어를 실행할 때 **매번 물어봅니다**.

`Claude wants to edit: src/App.tsx
Allow? (y/n)

Claude wants to execute: npm install axios
Allow? (y/n)`
**이런 분에게 추천합니다:**

- Claude Code를 처음 써보는 분

- 중요한 프로젝트에서 작업하는 분

- “내가 확인하고 넘어가야 안심된다”는 분

장점은 안전합니다. 단점은 파일 10개를 수정하면 10번 물어봅니다.

### 2. 보통 모드

**파일 읽기와 수정은 자동으로** 하고, 터미널 명령어(bash) 실행만 물어봅니다.

`✓ Claude edited: src/App.tsx (자동 허용)
✓ Claude edited: src/index.tsx (자동 허용)

Claude wants to execute: npm run build
Allow? (y/n)  ← 이것만 물어봄`
**이런 분에게 추천합니다:**

- Claude Code에 좀 익숙해진 분

- “파일 수정은 괜찮은데, 명령어 실행은 확인하고 싶다”는 분

- 여러 파일을 한 번에 수정하는 작업이 많은 분

> ℹ️ **정보**
>
> 가장 많이 쓰는 모드입니다. 파일 수정은 Git으로 되돌릴 수 있으니까 자동으로 하고, 시스템에 영향을 줄 수 있는 명령어 실행만 확인합니다.

### 3. 자유 모드 (YOLO 모드)

**모든 작업을 자동으로 진행합니다**. 물어보지 않습니다.

`✓ Claude edited: src/App.tsx
✓ Claude edited: src/index.tsx
✓ Claude executed: npm run build
✓ Claude executed: npm test`
물어보는 게 하나도 없으니 작업이 빠릅니다.

**이런 분에게 추천합니다:**

- Claude Code를 충분히 써봐서 동작 방식을 잘 아는 분

- Git으로 관리되는 프로젝트에서 작업하는 분

- 연습/실험 프로젝트에서 빠르게 작업하고 싶은 분

> ⚠️ **주의**
>
> 자유 모드는 편하지만, Claude가 의도하지 않은 명령어를 실행할 수도 있습니다. 반드시 Git이 설정된 프로젝트에서 사용하세요. 그래야 문제가 생겨도 되돌릴 수 있습니다.

## 모드 전환하기

### Shift+Tab으로 전환

Claude Code 안에서 **Shift+Tab**을 누르면 권한 모드를 전환할 수 있습니다.

`> (여기서 Shift+Tab 누르기)`
모드 선택 화면이 나타나고, 원하는 모드를 고르면 됩니다.

### 시작할 때 모드 지정하기

Claude Code를 처음 실행할 때 모드를 지정할 수도 있습니다.

`# 기본 모드 (신중한 모드)
claude
 
# 자유 모드로 시작
claude --dangerously-skip-permissions`

## 처음에는 어떤 모드가 좋을까요?

단계별로 추천합니다.

| 단계 | 추천 모드 | 이유 |
| --- | --- | --- |
| Day 1~2 (입문) | 신중한 모드 | 뭘 하는지 눈으로 확인하면서 배우기 |
| Day 3~4 (적응) | 보통 모드 | 파일 수정은 자동으로, 명령어만 확인 |
| Day 5~ (숙련) | 본인 선택 | 편한 모드로 전환 |

**지금 이 플레이북을 따라하고 있다면, 보통 모드를 추천합니다.** 파일 수정할 때마다 y를 누르는 건 번거롭고, Git이 있으니 되돌릴 수 있습니다.

---

## 권한 허용 범위 세부 설정

보통 모드에서도, 특정 명령어는 자동 허용하도록 세부 설정이 가능합니다.

예를 들어, `npm test`는 매번 물어보지 않게 하고 싶다면:

`Claude wants to execute: npm test
Allow? (y: once / a: always for this command / n: deny)`
**a를 누르면** 해당 명령어는 다음부터 자동으로 허용됩니다.

자주 쓰는 안전한 명령어들은 이렇게 등록해두면 편합니다:

- `npm test` — 테스트 실행

- `npm run build` — 빌드

- `npm run dev` — 개발 서버 실행

## 실습: 모드 전환해보기

**Step 1**: Claude Code를 실행합니다.

`claude`
**Step 2**: 아무 작업을 시켜봅니다.

`> README.md에 한 줄 추가해줘`
권한을 물어보면 y를 누릅니다.

**Step 3**: Shift+Tab을 눌러서 모드를 전환합니다.

보통 모드로 바꿔봅니다.

**Step 4**: 같은 작업을 다시 시켜봅니다.

`> README.md에 한 줄 더 추가해줘`
이번에는 파일 수정에 대해 물어보지 않고 자동으로 진행하는 걸 확인합니다.

## 자주 묻는 질문

**Q: 잘못 허용해서 파일이 망가지면 어떡하나요?**

Git이 있으면 됩니다. “방금 변경 되돌려줘”라고 Claude에게 말하면 됩니다. 앞 페이지에서 배운 내용이죠.

**Q: 자유 모드에서 Claude가 컴퓨터를 망가뜨릴 수 있나요?**

Claude Code는 기본적으로 위험한 시스템 명령어를 실행하지 않도록 설계되어 있습니다. 하지만 100% 안전을 보장하진 않으니, 중요한 프로젝트에서는 보통 모드 이상을 쓰는 게 좋습니다.

**Q: 모드를 바꾸면 이전 대화에 영향이 있나요?**

없습니다. 모드는 **앞으로의 작업**에만 적용됩니다.

- **Shift+Tab**으로 언제든 모드를 전환할 수 있습니다

- 처음에는 **보통 모드**를 추천합니다 (파일 수정 자동 + 명령어 확인)

- Git과 함께 쓰면 어떤 모드든 안전합니다
