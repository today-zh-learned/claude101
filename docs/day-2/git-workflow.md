# Git으로 협업하기

## 오늘의 목표

> Claude Code와 Git을 함께 써서 안전하게 코드를 관리합니다.

Claude Code가 파일을 수정하다가 뭔가 잘못되면 어떡하죠? 걱정하지 마세요. **Git**이 있으면 언제든 되돌릴 수 있습니다.

---

## Git이 뭔가요?

문서 작업을 하다가 실수했을 때, `Ctrl+Z`를 눌러서 되돌리죠? Git은 **프로젝트 전체에 대한 Ctrl+Z**입니다.

좀 더 정확히 말하면, Git은 **파일의 변경 이력을 기록하는 도구**입니다. 언제, 어떤 파일이, 어떻게 바뀌었는지 전부 기록합니다.

| 일반 작업 | Git으로 하면 |
| --- | --- |
| 파일을 덮어쓰면 이전 내용이 사라짐 | 모든 버전이 기록됨 |
| ”최종_최종_진짜최종.docx” | 깔끔한 변경 이력 |
| 실수하면 되돌리기 어려움 | `git checkout`으로 즉시 복구 |
| 누가 뭘 바꿨는지 모름 | 변경 기록이 남음 |

Claude Code와 함께 쓸 때 특히 중요합니다. Claude가 파일 10개를 한 번에 수정했는데 결과가 마음에 안 들면, Git으로 **한 번에 되돌릴 수 있습니다**.

> ⚠️ **주의**
>
> Git을 모르더라도 괜찮습니다. Claude Code가 Git 작업 대부분을 대신 해줍니다. 여기서는 “이런 개념이구나” 정도만 잡으면 됩니다.

---

## Claude Code가 알아서 해주는 Git 작업들

Claude Code는 Git을 잘 알고 있습니다. 이런 것들을 대신 해줍니다.

- **커밋 만들기**: 변경사항을 저장하고 메시지를 자동으로 작성합니다

- **변경 내역 확인**: “아까 뭘 바꿨는지 보여줘”라고 하면 보여줍니다

- **되돌리기**: “방금 변경 취소해줘”라고 하면 되돌려줍니다

- **브랜치 만들기**: “새 브랜치에서 작업하자”라고 하면 만들어줍니다

- **PR 만들기**: “PR 만들어줘”라고 하면 GitHub에 Pull Request를 생성합니다

여러분이 Git 명령어를 외울 필요가 없습니다. **한국어로 시키면 됩니다**.

---

## /commit으로 변경사항 저장하기

파일을 수정한 뒤, 변경사항을 저장(커밋)하고 싶을 때 두 가지 방법이 있습니다.

### 방법 1: 슬래시 커맨드

`> /commit`
Claude가 변경된 파일들을 확인하고, **커밋 메시지를 자동으로 작성**해줍니다. 여러분은 메시지를 확인하고 승인만 하면 됩니다.

### 방법 2: 자연어로 요청

`> 지금까지 변경한 거 커밋해줘`
같은 결과입니다. 편한 쪽을 쓰면 됩니다.

> ℹ️ **정보**
>
> 커밋 메시지는 “무엇을 왜 바꿨는지” 기록하는 메모입니다. 나중에 이력을 볼 때 “아, 이때 이 작업을 했구나”를 알 수 있게 해줍니다. Claude는 변경 내용을 분석해서 적절한 메시지를 작성합니다.

---

## 커밋 메시지 자동 생성

직접 Git을 쓰면 커밋 메시지를 매번 고민해야 합니다. Claude Code는 변경 내용을 분석해서 자동으로 작성합니다.

예를 들어, 버튼 색상을 바꾸고 `/commit`을 실행하면:

`커밋 메시지: "메인 페이지 CTA 버튼 색상을 파란색에서 초록색으로 변경"`
이런 식으로 구체적인 메시지를 만들어줍니다. 물론 마음에 안 들면 수정할 수 있습니다.

`> 커밋해줘. 메시지는 "버튼 스타일 업데이트"로 해줘`
---

## 브랜치 만들기와 PR 생성

브랜치는 **실험용 복사본**이라고 생각하면 됩니다. 원본은 그대로 두고, 복사본에서 마음껏 수정하는 겁니다.

마음에 들면 원본에 합치고, 마음에 안 들면 복사본을 버리면 됩니다.

### 브랜치 만들기

`> 새 브랜치 만들어줘. 이름은 feature/add-login-page로 해줘`
### 작업 후 PR 만들기

작업이 끝나면 PR(Pull Request)을 만들어서 원본에 합칠 수 있습니다.

`> PR 만들어줘`
Claude가 변경 내용을 분석해서 PR 제목과 설명을 자동으로 작성합니다. GitHub에 바로 올라갑니다.

> ℹ️ **정보**
>
> 혼자 작업하는데 브랜치가 왜 필요하냐고요? Claude Code에게 큰 작업을 시킬 때, 브랜치에서 하면 마음에 안 들 경우 **통째로 버릴 수 있습니다**. 안전장치입니다.

## 실습: 파일 수정 -> 커밋 -> 히스토리 확인

직접 해봅시다.

**Step 1**: 프로젝트 폴더에서 Claude Code를 실행합니다.

`cd ~/my-project
claude`
**Step 2**: 파일을 하나 수정합니다.

`> README.md에 "이 프로젝트는 연습용입니다"라는 줄을 추가해줘`
**Step 3**: 변경사항을 확인합니다.

`> 뭐가 바뀌었는지 보여줘`
Claude가 `git diff`를 실행해서 변경 내용을 보여줍니다.

**Step 4**: 커밋합니다.

`> /commit`
자동 생성된 커밋 메시지를 확인하고 승인합니다.

**Step 5**: 히스토리를 확인합니다.

`> 최근 커밋 이력 보여줘`
방금 만든 커밋이 기록되어 있는 걸 확인할 수 있습니다.

**Step 6** (선택): 되돌려봅니다.

`> 방금 커밋 되돌려줘`
되돌리기도 Claude에게 시키면 됩니다.

## 자주 쓰는 Git 관련 요청

Git을 몰라도 이렇게 말하면 됩니다.

| 하고 싶은 것 | Claude에게 이렇게 말하세요 |
| --- | --- |
| 변경사항 저장 | ”커밋해줘” 또는 `/commit` |
| 변경 내역 보기 | ”뭐가 바뀌었는지 보여줘” |
| 되돌리기 | ”방금 변경 취소해줘” |
| 새 브랜치 | ”새 브랜치 만들어줘” |
| PR 만들기 | ”PR 만들어줘” |
| 커밋 이력 | ”최근 커밋 이력 보여줘” |
| 특정 파일 복구 | ”index.html을 이전 버전으로 돌려줘” |

---

## 정리

- Git은 **프로젝트 전체의 Ctrl+Z**입니다

- Claude Code가 Git 작업 대부분을 대신 해줍니다

- `/commit`으로 변경사항을 저장하고, 커밋 메시지도 자동 생성됩니다

- 브랜치로 안전하게 실험하고, PR로 합칠 수 있습니다

- Git 명령어를 몰라도 **한국어로 시키면** Claude가 처리합니다
