# 실수해도 괜찮아 (체크포인트)

## 오늘의 목표

> 체크포인트로 언제든 이전 상태로 돌아갈 수 있다는 안심 얻기

코드를 수정하다 뭔가 잘못되면 어떡하죠? 걱정 마세요. Claude Code에는 **되돌리기 기능**이 내장되어 있습니다.

---

## 체크포인트란?

게임을 하다 보면 **세이브 포인트**가 있죠. 보스전에서 지면 세이브한 곳부터 다시 시작할 수 있습니다.

체크포인트가 바로 그겁니다. Claude Code가 파일을 수정할 때마다 **자동으로 현재 상태를 저장**해둡니다. 문제가 생기면 그 지점으로 돌아갈 수 있습니다.

`작업 흐름:

파일 수정 ← 체크포인트 1 자동 저장
    ↓
또 수정   ← 체크포인트 2 자동 저장
    ↓
뭔가 잘못됨!
    ↓
체크포인트 2로 복원 → 문제 해결`
---

## Claude가 자동으로 만드는 체크포인트

중요한 점은, 여러분이 **아무것도 하지 않아도** 체크포인트가 만들어진다는 것입니다.

Claude Code는 다음 상황에서 자동으로 체크포인트를 생성합니다:

| 상황 | 체크포인트 생성 |
| --- | --- |
| 파일 수정 (Write/Edit) | 수정 전 상태를 저장 |
| 파일 생성 | 생성 전 상태를 저장 |
| 명령어 실행 (Bash) | 실행 전 상태를 저장 |

별도의 설정이나 명령어가 필요 없습니다. 그냥 Claude와 대화하면서 작업하면, 뒤에서 알아서 체크포인트가 쌓입니다.

> ℹ️ **정보**
>
> 체크포인트는 **Git과 별개**로 동작합니다. Git 커밋을 하지 않았어도 체크포인트로 복원할 수 있습니다.

---

## 돌아가기: Esc 두 번

작업 중 “방금 한 거 되돌리고 싶다”는 생각이 들면:

1. **Esc 키를 두 번** 누릅니다

1. Claude가 최근 변경 사항을 보여줍니다

1. 복원할지 선택합니다

`> Esc Esc

Claude: 마지막 작업에서 다음 파일이 변경되었습니다:
  - src/index.js (수정됨)
  - src/utils.js (새로 생성)

복원하시겠습니까?`
이때 복원을 선택하면 **변경 전 상태로** 돌아갑니다. 방금 만든 파일도 사라지고, 수정한 내용도 원래대로 돌아옵니다.

---

## /rewind로 특정 시점으로 돌아가기

Esc 두 번은 **바로 직전**으로 돌아가는 것입니다. 만약 “3단계 전으로 돌아가고 싶다”면 `/rewind` 명령어를 씁니다.

`> /rewind`
이렇게 입력하면 Claude가 **대화 기록을 타임라인으로** 보여줍니다:

`대화 기록:
  1. [10:01] 파일 구조 설명
  2. [10:03] index.js 수정       ← 체크포인트
  3. [10:05] utils.js 생성       ← 체크포인트
  4. [10:08] 테스트 실행
  5. [10:10] index.js 재수정     ← 체크포인트

어디로 돌아가시겠습니까?`
원하는 시점의 번호를 선택하면 그 시점으로 돌아갑니다.

> ⚠️ **주의**
>
> `/rewind`로 돌아가면 그 이후의 대화와 변경 사항이 모두 사라집니다. 신중하게 선택하세요.

---

## 체크포인트 vs Git: 뭐가 다른지

둘 다 “이전으로 돌아가기” 기능인데, 뭐가 다를까요?

| 항목 | 체크포인트 | Git |
| --- | --- | --- |
| **생성 방식** | 자동 | 수동 (`git commit`) |
| **범위** | 현재 대화 안에서만 | 영구 보존 |
| **용도** | 작업 중 실수 되돌리기 | 버전 관리, 협업 |
| **대화 종료 후** | 사라짐 | 유지됨 |
| **비유** | 게임 중 퀵세이브 | 게임 완료 후 정식 세이브 |

정리하면:

- **체크포인트**: 작업하면서 자유롭게 되돌리는 용도 (임시)

- **Git**: 완성된 작업을 기록하는 용도 (영구)

> ℹ️ **정보**
>
> 체크포인트는 대화가 끝나면 사라집니다. 중요한 변경 사항은 반드시 **Git 커밋으로 저장**하세요. Claude에게 “커밋해줘”라고 말하면 됩니다.

## 실습: 일부러 망가뜨리고 복원해보기

직접 해보는 게 가장 빠릅니다. 연습 프로젝트에서 이 순서를 따라해보세요.

### Step 1: 파일 만들기

`> hello.js 파일 만들어줘. console.log("안녕하세요")를 출력하는 코드로.`
Claude가 `hello.js` 파일을 만듭니다. 체크포인트 1이 자동 생성됩니다.

### Step 2: 일부러 망가뜨리기

`> hello.js를 수정해줘. console.log를 전부 지우고 에러나는 코드로 바꿔줘.`
Claude가 파일을 망가뜨립니다. 체크포인트 2가 자동 생성됩니다.

### Step 3: 복원하기

`Esc Esc`
Esc를 두 번 누르면 체크포인트 2로 돌아가서, 망가뜨리기 전 상태인 원래 `hello.js`가 복원됩니다.

### Step 4: 확인하기

`> hello.js 내용 보여줘`
원래 코드가 돌아와 있습니다. “안녕하세요”가 정상적으로 출력되는 코드로 복구된 걸 확인할 수 있습니다.

> ℹ️ **정보**
>
> 이 실습을 한 번 해보면 마음이 편해집니다. “잘못되면 어쩌지”라는 걱정 없이 대담하게 실험할 수 있게 됩니다.

---

## 체크포인트 활용 팁

실전에서 체크포인트를 잘 쓰는 방법 몇 가지를 공유합니다:

1. **실험적인 시도 전에 Git 커밋부터**: “이거 될까?” 싶은 실험을 하기 전에, 현재 상태를 Git으로 커밋해두세요. 체크포인트 + Git 이중 보호가 됩니다.

1. **대화가 길어지면 중간 커밋**: 체크포인트는 대화가 끝나면 사라집니다. 중간중간 “여기까지 커밋해줘”라고 말해두세요.

1. **복원 후에도 다시 시도 가능**: 복원했다고 해서 끝이 아닙니다. 다른 방법으로 다시 시도하면 됩니다. 체크포인트가 또 생기니까요.

---

## 정리

오늘 배운 것을 정리합니다:

- 체크포인트는 Claude가 파일을 수정할 때 **자동으로 생성**됩니다

- **Esc 두 번**으로 바로 직전 상태로 돌아갈 수 있습니다

- **`/rewind`**로 특정 시점을 골라 돌아갈 수 있습니다

- 체크포인트는 대화 중 임시 저장, Git은 영구 저장입니다

- 실수를 두려워하지 마세요 — 언제든 되돌릴 수 있습니다
