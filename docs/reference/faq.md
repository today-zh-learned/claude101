# 자주 묻는 질문

> Claude Code에 대해 자주 묻는 질문 20개를 모았습니다.

---

## 일반

### Claude Code는 무료인가요?

완전 무료는 아닙니다. 두 가지 방식이 있습니다.

- **Anthropic API 사용**: API 키를 발급받아 사용량만큼 비용을 지불합니다. Claude Code 도구 자체는 무료이고, API 호출 비용만 발생합니다.

- **Claude Pro/Max 구독**: 월 구독료에 Claude Code 사용량이 포함되어 있습니다. Pro는 제한적, Max는 넉넉한 사용량을 제공합니다.

### 인터넷 연결이 필요한가요?

네, 필수입니다. Claude Code는 Anthropic의 API 서버와 통신해서 동작합니다. 로컬에서 파일을 읽고 쓰는 건 오프라인에서도 되지만, Claude에게 질문하거나 응답을 받으려면 인터넷이 연결되어 있어야 합니다.

### 어떤 프로그래밍 언어를 지원하나요?

특정 언어에 제한이 없습니다. Python, JavaScript, TypeScript, Java, Go, Rust, C/C++, Ruby, PHP, Swift, Kotlin 등 주요 언어를 모두 다룹니다. HTML/CSS, SQL, YAML, JSON 같은 비프로그래밍 언어도 잘 처리합니다.

Claude는 프로그래밍 언어뿐 아니라 설정 파일, 셸 스크립트, 문서 작성까지 폭넓게 도와줍니다.

### 내 코드가 안전한가요?

Claude Code는 코드를 Anthropic API로 전송해서 분석합니다. Anthropic의 정책에 따르면 **API를 통해 전송된 데이터는 모델 학습에 사용되지 않습니다.** 다만 일정 기간 보관될 수 있으므로, 극도로 민감한 코드(의료 정보, 금융 시스템 핵심 로직 등)는 주의가 필요합니다.

> ℹ️ **정보**
>
> 회사에서 사용한다면, 보안팀에 Anthropic의 데이터 처리 정책을 확인시켜주세요. 대부분의 경우 API 사용은 학습 데이터에 포함되지 않습니다.

### Cursor랑 뭐가 다른가요?

Cursor는 VS Code 기반의 **GUI 에디터**이고, Claude Code는 **터미널(CLI) 기반 도구**입니다.

| 항목 | Claude Code | Cursor |
| --- | --- | --- |
| 인터페이스 | 터미널 (CLI) | VS Code 기반 에디터 |
| 작업 방식 | 대화로 지시 | 에디터 내 AI 기능 |
| 자동화 | 셸 명령 직접 실행 | 에디터 작업 위주 |
| 파일 편집 | 여러 파일 동시 수정 | 현재 파일 중심 |
| 비용 | API 사용량 기반 | 월 구독 |
| 장점 | 자동화, Git, 대규모 작업 | 시각적 편집, 직관적 |

두 도구는 경쟁이 아니라 **보완 관계**입니다. Cursor로 눈으로 보면서 편집하고, Claude Code로 자동화와 대규모 작업을 처리하는 조합도 좋습니다.

## 비용

### 한 달에 얼마 정도 드나요?

사용량에 따라 다르지만, 일반적인 패턴은 이렇습니다.

| 사용 패턴 | 예상 월 비용 (API) | 구독 대안 |
| --- | --- | --- |
| 가벼운 사용 (하루 10~20회) | $5~15 | Pro ($20/월) |
| 보통 사용 (하루 30~50회) | $20~50 | Pro ($20/월) |
| 헤비 사용 (하루 100회+) | $50~150+ | Max ($100/월 또는 $200/월) |

`/cost` 명령어로 현재 세션 비용을 실시간 확인할 수 있습니다.

### 완전 무료로 쓸 수 있는 방법이 있나요?

Anthropic에서 신규 가입 시 제공하는 무료 크레딧으로 체험해볼 수 있습니다. 또한 Claude Code 자체는 오픈소스(Apache 2.0)이므로 설치는 무료입니다. 다만 실제 사용하려면 API 키(유료) 또는 구독이 필요합니다.

AWS Bedrock이나 Google Vertex를 이미 사용하고 있다면 해당 플랫폼의 크레딧으로 Claude Code를 쓸 수도 있습니다.

### Max 플랜이란 뭔가요?

Claude Max는 Anthropic의 월 구독 플랜 중 상위 플랜입니다.

- **Max $100/월**: 일반 사용에 넉넉한 양. Sonnet 모델 위주 사용.

- **Max $200/월**: 대량 사용, Opus 모델까지 여유 있게 사용.

Max 플랜의 장점은 **API 키 없이** 로그인만으로 사용 가능하고, 매달 정해진 비용이라 예산 관리가 쉽다는 점입니다.

### 비용 제한을 설정할 수 있나요?

네, Anthropic Console에서 월간 사용 한도(spend limit)를 설정할 수 있습니다. 한도에 도달하면 API 호출이 차단되어 예상치 못한 과금을 방지할 수 있습니다.

`# 세션별 비용 확인
/cost
 
# Anthropic Console에서 설정
# https://console.anthropic.com → Settings → Limits`

## 기능

### 이미지를 분석할 수 있나요?

네, 가능합니다. Claude Code는 이미지 파일을 읽을 수 있습니다.

`이 스크린샷 분석해줘: screenshot.png`
UI 디자인 스크린샷을 보여주고 해당 디자인을 구현해달라고 요청하거나, 에러 화면 스크린샷으로 문제를 진단받을 수도 있습니다.

### 웹 검색을 할 수 있나요?

Claude Code 자체에는 웹 검색 기능이 없습니다. 하지만 **MCP 서버**를 통해 웹 검색 기능을 추가할 수 있습니다.

`# 웹 검색 MCP 서버 추가 예시
claude mcp add web-search npx @anthropic-ai/mcp-server-web-search`
MCP 설정에 대한 자세한 내용은 [Day 5 - MCP]({{ '/docs/day-5/mcp.html' | relative_url }})를 참고하세요.

### 파일 업로드가 가능한가요?

Claude Code는 터미널에서 동작하므로, 파일 “업로드”라는 개념보다는 **파일 경로를 알려주는** 방식입니다. 프로젝트 폴더 안에 있는 파일은 모두 읽을 수 있습니다.

`src/data/users.csv 파일을 분석해줘
docs/design.png을 보고 컴포넌트 만들어줘`
프로젝트 외부 파일은 `--add-dir` 옵션으로 추가 디렉토리를 지정하면 됩니다.

### 여러 프로젝트를 동시에 작업할 수 있나요?

네. 터미널 탭이나 창을 여러 개 열어서 각각 다른 폴더에서 Claude Code를 실행하면 됩니다. 각 세션은 독립적으로 동작합니다.

`# 터미널 1
cd ~/project-a && claude
 
# 터미널 2
cd ~/project-b && claude`
다만, 같은 API 키를 사용하므로 사용량은 합산됩니다.

### 오프라인에서 쓸 수 있나요?

아니요. Claude Code는 Anthropic API와 통신해야 하므로 인터넷 연결이 필수입니다. 오프라인에서는 이전 대화를 확인하거나 로컬 파일을 다루는 정도만 가능하고, 새로운 질문이나 코드 생성은 불가합니다.

## 보안

### 내 코드가 AI 학습에 사용되나요?

**API를 통한 사용은 학습에 사용되지 않습니다.** 이것은 Anthropic의 공식 정책입니다. API 사용약관에 명시되어 있으며, 기업 고객도 안심하고 사용할 수 있는 이유입니다.

다만 **claude.ai 웹 무료 버전**에서의 대화는 학습에 사용될 수 있으므로, 민감한 코드는 웹 버전에 붙여넣지 마세요.

### API 키는 어디에 저장되나요?

API 키는 다음 위치에 저장됩니다.

| 방식 | 저장 위치 | 보안 수준 |
| --- | --- | --- |
| 환경변수 | `~/.zshrc` 또는 `~/.bashrc` | 보통 |
| Claude 로그인 | `~/.claude/credentials.json` | 보통 |
| 시스템 키체인 | OS 키체인 (macOS Keychain 등) | 높음 |

> ⚠️ **주의**
>
> `.env` 파일에 API 키를 저장한다면, 반드시 `.gitignore`에 `.env`를 추가하세요. Git에 키가 커밋되면 큰 보안 사고로 이어질 수 있습니다.

### 팀 환경에서 보안은 어떻게 관리하나요?

팀에서 사용할 때 권장 사항입니다.

1. **각자 개인 API 키 사용** — 팀 공용 키는 사용하지 마세요

1. **`.claude/settings.json`의 권한 설정** — 위험한 명령을 차단

1. **CLAUDE.md에 보안 규칙 명시** — “`.env` 파일을 절대 수정하지 마세요” 등

1. **Hooks로 가드레일 설정** — `PreToolUse` 훅으로 특정 파일 수정 차단

`// .claude/settings.json 예시
{
  "permissions": {
    "deny": ["Bash(rm -rf *)", "Edit(.env)"]
  }
}`

## 문제 상황

### Claude가 잘못된 답변을 했을 때는?

AI는 완벽하지 않습니다. 잘못된 코드를 생성하거나 틀린 설명을 할 수 있습니다.

**대처법**:

1. **되돌리기** — Claude가 파일을 잘못 수정했다면 `git checkout -- 파일명`으로 복원

1. **구체적으로 지적** — “이 부분이 틀렸어. 이유는 …”이라고 알려주면 수정합니다

1. **검증 요청** — “이 코드가 정말 맞는지 테스트 돌려봐”라고 요청

1. **새 세션** — 대화가 꼬였다면 `/clear` 후 처음부터 다시 설명

> ℹ️ **정보**
>
> Claude가 생성한 코드는 항상 검토하세요. 특히 보안 관련 코드(인증, 암호화 등)는 반드시 직접 확인하시기 바랍니다.

### Claude가 너무 느릴 때는?

응답 속도가 느린 원인과 해결법입니다.

| 원인 | 해결법 |
| --- | --- |
| 프로젝트가 큼 | 작업할 파일을 구체적으로 지정 |
| 복잡한 요청 | 작업을 작게 나눠서 요청 |
| 서버 부하 | 잠시 기다렸다 재시도 |
| 무거운 모델 | `/model`로 가벼운 모델로 전환 |
| 대화가 김 | `/compact`로 압축 |
| 네트워크 | VPN 끄기, Wi-Fi 확인 |

### Claude가 코드를 망가뜨렸을 때는?

당황하지 마세요. Git을 쓰고 있다면 간단히 복구됩니다.

`# 변경사항 확인
git diff
 
# 특정 파일만 되돌리기
git checkout -- src/broken-file.ts
 
# 모든 변경 되돌리기 (주의!)
git checkout -- .
 
# 이전 커밋으로 되돌리기
git log --oneline  # 커밋 확인
git revert HEAD    # 최근 커밋 취소 (안전)`
> ℹ️ **정보**
>
> 이것이 Claude Code를 **항상 Git 프로젝트 안에서** 사용해야 하는 이유입니다. Git이 있으면 어떤 실수도 되돌릴 수 있습니다. [Day 2 - Git 워크플로우]({{ '/docs/day-2/git-workflow.html' | relative_url }})에서 자세히 다룹니다.
