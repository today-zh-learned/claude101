# 모델 골라 쓰기

## 오늘의 목표

> 상황에 맞는 AI 모델을 골라서 비용과 성능을 최적화합니다.

Claude Code를 쓰다 보면, 어떤 작업은 “이 정도에 이렇게 비싼 모델을 쓸 필요가 있나?” 싶고, 어떤 작업은 “좀 더 똑똑한 모델이 필요한데” 싶을 때가 있습니다. Claude에는 **여러 모델**이 있고, 작업 중간에 자유롭게 전환할 수 있습니다.

---

## Claude 모델 종류

Anthropic에서 제공하는 Claude 모델은 크게 세 가지 티어로 나뉩니다.

| 모델 | 특징 | 비용 | 속도 |
| --- | --- | --- | --- |
| **Opus 4.6** | 가장 똑똑함. 복잡한 추론, 창의적 작업에 강함. 1M 컨텍스트 | 높음 | 느림 |
| **Sonnet 4.6** | 균형 잡힌 성능. 대부분의 코딩 작업에 충분. 200K 컨텍스트 | 중간 | 중간 |
| **Haiku 4.5** | 빠르고 저렴. 단순 작업에 적합 | 낮음 | 빠름 |

Claude Code의 기본 모델은 **Sonnet**입니다. 대부분의 코딩 작업에서 충분한 성능을 보여주기 때문입니다.

> ℹ️ **정보**
>
> 2026년 3월 기준 최신 모델: Opus 4.6 (`claude-opus-4-6`), Sonnet 4.6 (`claude-sonnet-4-6`), Haiku 4.5 (`claude-haiku-4-5-20251001`). Opus 4.6은 1M(100만) 토큰 컨텍스트를 지원하여 대규모 프로젝트 분석에 특히 강력합니다.

---

## 언제 어떤 모델을 쓰나요?

모든 작업에 Opus를 쓰면 좋겠지만, 비용이 부담됩니다. 반대로 모든 걸 Haiku로 하면 품질이 떨어집니다. **작업 성격에 맞게 모델을 선택**하는 것이 핵심입니다.

| 작업 유형 | 추천 모델 | 이유 |
| --- | --- | --- |
| 시스템 아키텍처 설계 | **Opus** | 복잡한 구조를 정확히 파악하고 설계해야 함 |
| 멀티파일 리팩토링 | **Opus** | 파일 간 의존성을 이해하고 일관성 유지 필요 |
| 일반 기능 구현 | **Sonnet** | 대부분의 코딩에 충분한 성능 |
| 버그 수정 | **Sonnet** | 코드 분석 + 수정에 균형 잡힌 선택 |
| 파일 이름 변경 | **Haiku** | 단순 반복 작업, 빠르게 처리 |
| 주석/문서 추가 | **Haiku** | 패턴이 명확한 작업 |
| 코드 포매팅 | **Haiku** | 규칙 기반 작업 |

한 가지 팁을 드리면, **먼저 Sonnet으로 시도**하고 결과가 만족스럽지 않을 때 Opus로 올리는 방식이 실용적입니다.

---

## /model 명령어로 전환하기

Claude Code 대화 중에 `/model` 명령어로 모델을 전환할 수 있습니다.

`> /model claude-opus-4-6`
현재 사용 중인 모델을 확인하려면 그냥 `/model`만 입력합니다.

`> /model`
사용 가능한 모델 목록이 표시되고 선택할 수 있습니다.

실제 작업 흐름에서는 이렇게 쓸 수 있습니다.

`> /model opus
> 이 프로젝트의 전체 아키텍처를 분석하고 개선점을 제안해줘

(분석 완료 후)

> /model sonnet
> 방금 제안한 개선점 중 1번을 구현해줘

(구현 완료 후)

> /model haiku
> 새로 만든 파일들에 JSDoc 주석을 추가해줘`
하나의 세션에서 작업 단계마다 모델을 바꾸는 것이 자연스럽습니다.

## 서브에이전트에 다른 모델 지정하기

Claude Code에서 **서브에이전트**(Task 도구로 호출되는 하위 에이전트)를 사용할 때, 메인 에이전트와 다른 모델을 지정할 수 있습니다.

예를 들어, 메인 작업은 Opus로 진행하면서 단순 하위 작업은 Haiku로 처리하는 식입니다.

`메인 에이전트 (Opus): 전체 설계 + 판단
  └─ 서브에이전트 (Haiku): 파일 검색, 단순 변환`
이 부분은 Day 6에서 에이전트 시스템을 다룰 때 자세히 실습합니다. 지금은 “서브에이전트별로 모델을 다르게 쓸 수 있다”는 것만 기억해두면 됩니다.

> ℹ️ **정보**
>
> 서브에이전트 모델 지정은 CLAUDE.md나 에이전트 설정 파일에서 할 수 있습니다. Day 6에서 구체적인 설정법을 다룹니다.

## 비용 비교

모델별 비용 차이를 체감하기 위해 대략적인 비교를 해봅시다.

| 모델 | 입력 토큰 (100만) | 출력 토큰 (100만) | 상대 비용 |
| --- | --- | --- | --- |
| **Opus 4.6** | $15 | $75 | 5x |
| **Sonnet 4.6** | $3 | $15 | 1x (기준) |
| **Haiku 4.5** | $0.80 | $4 | 0.27x |

같은 작업을 Opus로 하면 Sonnet 대비 약 5배, Haiku 대비 약 19배의 비용이 듭니다.

> ⚠️ **주의**
>
> Max 요금제를 사용하면 토큰 단가 대신 월 정액으로 사용할 수 있습니다. 이 경우에도 모델별로 사용량 한도가 다르므로, 적절한 모델 선택은 여전히 중요합니다.

실제로는 이런 전략이 효과적입니다.

- **일상 작업의 80%**: Sonnet으로 처리

- **복잡한 설계/디버깅 10%**: Opus로 전환

- **단순 반복 작업 10%**: Haiku로 처리

이렇게 하면 Opus만 쓸 때 대비 비용을 크게 줄이면서도, 필요한 곳에서는 최고 성능을 활용할 수 있습니다.

---

## 정리

- Claude는 **Opus**(고성능), **Sonnet**(균형), **Haiku**(경제적) 세 가지 모델 티어가 있습니다

- `/model` 명령어로 대화 중 언제든 전환할 수 있습니다

- 작업 성격에 맞는 모델을 선택하면 **비용 대비 효율**이 크게 올라갑니다

- 서브에이전트에 별도 모델을 지정할 수 있습니다 (Day 6에서 자세히)

- 기본 전략: Sonnet을 기본으로, 필요할 때 Opus/Haiku로 전환
