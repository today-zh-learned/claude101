# 웹에서도 쓸 수 있어요

## 오늘의 목표

> 다양한 환경에서 Claude Code를 사용하는 방법을 알아봅니다.

지금까지는 로컬 터미널에서 Claude Code를 사용했습니다. 하지만 항상 내 컴퓨터 앞에 앉아 있을 수는 없습니다. 카페에서 태블릿만 들고 있을 때, 출장 중에 호텔 컴퓨터밖에 없을 때, 팀원에게 환경 설치 없이 체험하게 해주고 싶을 때 — 이런 상황에서도 Claude Code를 쓸 수 있는 방법들이 있습니다.

---

## claude.ai 웹에서 Claude Code 기능

[claude.ai](https://claude.ai) 웹 인터페이스에서도 Claude Code와 유사한 코딩 기능을 사용할 수 있습니다.

**Artifacts**: Claude가 코드를 작성하면 오른쪽 패널에 실시간 미리보기가 표시됩니다. HTML, React 컴포넌트, 다이어그램 등을 바로 확인할 수 있습니다.

**Projects**: 프로젝트를 만들고 파일을 업로드하면, 해당 파일들을 맥락으로 활용한 대화가 가능합니다. CLAUDE.md와 비슷한 역할을 합니다.

다만 웹 버전에는 제약이 있습니다.

| 기능 | claude.ai 웹 | Claude Code (CLI) |
| --- | --- | --- |
| 코드 생성 | 가능 | 가능 |
| 파일 직접 수정 | 불가 (복사 필요) | 가능 |
| 명령어 실행 | 불가 | 가능 |
| Git 연동 | 불가 | 가능 |
| MCP 연결 | 불가 | 가능 |

웹은 **빠른 프로토타이핑이나 코드 질문**에 적합합니다. 본격적인 프로젝트 작업은 CLI가 필요합니다.

---

## GitHub Codespaces에서 사용하기

**GitHub Codespaces**는 브라우저에서 VS Code 환경을 제공하는 서비스입니다. 여기에 Claude Code를 설치하면 **어떤 컴퓨터에서든** 브라우저만으로 Claude Code를 사용할 수 있습니다.

### 설정 방법

**Step 1**: GitHub 저장소에서 Code 버튼 > Codespaces 탭 > “Create codespace on main”을 클릭합니다.

**Step 2**: 터미널이 열리면 Claude Code를 설치합니다.

`npm install -g @anthropic-ai/claude-code`
**Step 3**: API 키를 설정합니다.

`export ANTHROPIC_API_KEY="sk-ant-..."`
**Step 4**: 실행합니다.

`claude`
이제 브라우저 안에서 Claude Code를 사용할 수 있습니다. 저장소의 모든 파일에 접근 가능하고, 터미널 명령어도 실행됩니다.

> ℹ️ **정보**
>
> Codespaces는 GitHub 무료 계정에도 월 120시간의 무료 사용 시간이 제공됩니다. 개인 프로젝트에는 충분한 양입니다.

## 원격 서버에서 사용하기

회사 서버나 클라우드 인스턴스(AWS EC2, GCP VM 등)에서도 Claude Code를 사용할 수 있습니다. SSH로 접속해서 동일하게 설치하면 됩니다.

`# 원격 서버에 SSH 접속
ssh user@my-server.com
 
# Node.js가 설치되어 있다면
npm install -g @anthropic-ai/claude-code
 
# API 키 설정
export ANTHROPIC_API_KEY="sk-ant-..."
 
# 실행
claude`
원격 서버에서 사용하면 이런 장점이 있습니다.

- **서버 환경에서 직접 작업**: 배포 환경과 동일한 곳에서 코드를 수정하고 테스트

- **고성능 하드웨어 활용**: 로컬 머신보다 강력한 서버에서 빌드나 테스트 실행

- **장시간 작업**: 로컬 컴퓨터를 꺼도 서버에서 작업이 계속됨 (tmux/screen 사용 시)

> ⚠️ **주의**
>
> 원격 서버에서 Claude Code를 쓸 때는 `tmux`나 `screen`을 사용하는 것을 권장합니다. SSH 연결이 끊어져도 Claude Code 세션이 유지됩니다.

### tmux 기본 사용법

`# 새 세션 시작
tmux new -s claude-session
 
# Claude Code 실행
claude
 
# 세션에서 빠져나오기 (Claude는 계속 실행됨)
# Ctrl+B, D
 
# 나중에 다시 연결
tmux attach -t claude-session`

## 모바일에서 접근하기

스마트폰이나 태블릿에서도 Claude Code를 사용할 수 있습니다. 직접 설치하는 것은 아니고, **SSH 앱으로 원격 서버에 접속**하는 방식입니다.

### 추천 SSH 앱

| 플랫폼 | 앱 | 특징 |
| --- | --- | --- |
| iOS | **Termius** | 깔끔한 UI, 키보드 커스터마이징 |
| iOS | **Blink Shell** | 전문가용, Mosh 지원 |
| Android | **Termux** | 로컬 터미널 + SSH 클라이언트 |
| Android | **JuiceSSH** | 간편한 SSH 클라이언트 |

모바일에서의 작업은 긴급한 수정이나 간단한 확인 용도로 적합합니다. 긴 코딩 세션에는 화면 크기와 키보드의 한계가 있습니다.

### 활용 시나리오

`1. 서버에 tmux + Claude Code를 미리 띄워둠
2. 외출 중 모바일 SSH 앱으로 접속
3. tmux 세션에 연결
4. "이 버그 수정해줘" 한 줄 입력
5. Claude가 수정하면 확인 후 승인`
일상적으로 쓰기보다는, **긴급 상황에서 어디서든 접근할 수 있다**는 점이 핵심입니다.

## 각 환경의 장단점

| 환경 | 장점 | 단점 | 추천 상황 |
| --- | --- | --- | --- |
| **로컬 터미널** | 가장 빠름, 풀 기능 | 내 컴퓨터 필요 | 일상 작업 |
| **claude.ai 웹** | 설치 불필요, 어디서든 접근 | 파일 조작/명령어 실행 불가 | 빠른 질문, 프로토타이핑 |
| **Codespaces** | 브라우저만 있으면 됨 | 약간의 지연, 무료 시간 제한 | 다른 컴퓨터에서 작업 |
| **원격 서버** | 고성능, 장시간 작업 | 서버 비용, 설정 필요 | 서버 환경 직접 작업 |
| **모바일 SSH** | 어디서든 접근 | 작은 화면, 불편한 키보드 | 긴급 수정 |

> ℹ️ **정보**
>
> 가장 실용적인 조합은 **로컬 터미널 (메인) + Codespaces (이동 시) + 모바일 SSH (긴급 시)**입니다. 세 가지를 준비해두면 어떤 상황에서든 Claude Code에 접근할 수 있습니다.

---

## 정리

- **claude.ai 웹**: 코드 질문과 프로토타이핑에 적합하지만 파일 직접 조작은 불가

- **GitHub Codespaces**: 브라우저에서 완전한 Claude Code 환경을 사용할 수 있음

- **원격 서버**: SSH + tmux로 장시간 작업과 고성능 환경 활용 가능

- **모바일 SSH**: 긴급 수정 시 어디서든 접근 가능

- 상황에 맞는 환경을 조합하면 장소에 구애받지 않고 작업할 수 있습니다
