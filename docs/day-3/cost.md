# 비용, 얼마나 드나요?

## 오늘의 목표

> Claude Code의 비용 구조를 이해하고, 현명하게 관리하는 법을 배웁니다.

AI 도구를 처음 쓸 때 가장 걱정되는 게 비용입니다. “혹시 모르는 사이에 요금 폭탄 맞는 건 아닐까?” 결론부터 말씀드리겠습니다.

일반적인 개인 사용 기준, **월 $10~30 수준**이면 충분합니다. Max 플랜을 쓰면 월 $100 또는 $200 정액제로 쓸 수 있습니다. 무서워할 금액이 아닙니다.

---

## 비용이 발생하는 구조

Claude Code는 **토큰** 단위로 비용이 책정됩니다. 토큰은 대략 “글자 수 비슷한 것”이라고 생각하면 됩니다.

비용이 발생하는 순간은 크게 두 가지입니다:

1. **입력 토큰**: 여러분이 보내는 프롬프트 + Claude가 읽는 파일 내용

1. **출력 토큰**: Claude가 생성하는 답변 + 코드

`비용 = (입력 토큰 수 × 입력 단가) + (출력 토큰 수 × 출력 단가)`
여기서 핵심은 **입력 토큰이 출력 토큰보다 훨씬 많다**는 겁니다. Claude에게 파일 10개를 읽으라고 하면 그것만으로 입력 토큰이 크게 늘어납니다. 앞에서 배운 컨텍스트 관리가 곧 비용 관리입니다.

---

## /cost로 현재 사용량 확인하기

대화 중에 언제든 `/cost`를 입력하면 현재 세션의 사용량을 확인할 수 있습니다.

`> /cost`
현재 대화에서 사용한 토큰 수와 예상 비용이 표시됩니다. 습관적으로 확인하면 “어떤 작업이 비용을 많이 먹는지” 감이 잡힙니다.

---

## 비용 줄이는 5가지 팁

### 1. /compact 자주 쓰기

앞에서 배운 `/compact`가 여기서도 빛을 발합니다. 컨텍스트를 압축하면 **다음 요청의 입력 토큰이 줄어들어** 비용이 절약됩니다.

### 2. 새 대화 적절히 시작하기

주제가 바뀌면 새 대화를 시작하세요. 이전 대화의 컨텍스트를 불필요하게 끌고 다니면 매 요청마다 토큰 낭비입니다.

### 3. CLAUDE.md 간결하게 유지

CLAUDE.md는 매 대화마다 로딩됩니다. 여기에 장문의 설명을 넣으면 **매번** 그 비용이 발생합니다. 핵심만 간결하게 적으세요.

### 4. 모델 선택하기

Claude Code에서는 모델을 선택할 수 있습니다.

| 모델 | 특징 | 추천 상황 |
| --- | --- | --- |
| **Sonnet** | 빠르고 저렴 | 간단한 작업, 파일 수정, 질문 답변 |
| **Opus** | 느리지만 똑똑 | 복잡한 설계, 대규모 리팩토링, 어려운 버그 |

간단한 작업에 Opus를 쓰는 건 택시비로 택배 보내는 것과 같습니다. **기본은 Sonnet, 어려운 작업만 Opus**가 가성비 좋은 전략입니다.

모델 전환은 `/model` 명령어로 합니다. “이건 좀 복잡한데…” 싶을 때만 Opus로 바꾸세요.

### 5. 불필요한 파일 읽기 줄이기

`❌ "src 폴더 전체 분석해줘"        → 파일 50개를 다 읽음
✅ "src/auth/login.ts 분석해줘"    → 파일 1개만 읽음`
**필요한 파일만 정확히 지정**하는 습관이 비용 절약의 핵심입니다.

| 모델 | 입력 (100만 토큰당) | 출력 (100만 토큰당) | 비고 |

위 가격은 API 직접 사용 기준입니다. Max 플랜을 쓰면 정액제이므로 토큰당 과금이 아닙니다. 자세한 내용은 아래 플랜 비교를 참고하세요.

## Max 플랜 vs API 플랜

Claude Code를 쓰는 방법은 크게 두 가지입니다.

### Max 플랜 (추천)

| 플랜 | 월 가격 | 포함 내용 |
| --- | --- | --- |
| Pro | $20/월 | Claude Code 기본 사용 가능 (사용량 제한 있음) |
| Max 5x | $100/월 | Pro 대비 5배 사용량 |
| Max 20x | $200/월 | Pro 대비 20배 사용량 |

**장점**: 정액제라 요금 폭탄 걱정이 없습니다. 매달 고정 비용만 나갑니다.
**단점**: 사용량 한도가 있어서, 많이 쓰면 속도가 제한될 수 있습니다.

### API 플랜

Anthropic API 키를 발급받아 사용하는 방식입니다.

**장점**: 사용한 만큼만 정확히 과금. 한도 없이 사용 가능.
**단점**: 사용량을 직접 모니터링해야 합니다. 실수로 많이 쓸 수 있습니다.

### 어떤 걸 선택할까?

| 상황 | 추천 |
| --- | --- |
| 처음 시작, 비용 걱정됨 | **Pro 플랜** ($20/월)으로 시작 |
| 매일 1~2시간 사용 | **Max 5x** ($100/월) |
| 하루 종일 개발에 사용 | **Max 20x** ($200/월) 또는 API |
| 팀에서 대량 사용 | **API 플랜** |

처음에는 **Pro 플랜($20/월)**으로 시작하세요. 써보다가 사용량이 부족하면 그때 업그레이드하면 됩니다. 한 달 써보면 본인에게 맞는 플랜이 자연스럽게 보입니다.

## 비용 대비 효과

“월 $20~200이 비싸지 않나요?”라는 질문을 자주 받습니다.

비교해봅시다. Claude Code로 30분 만에 끝내는 작업을 직접 하면 3~4시간 걸립니다. 여러분의 시급이 2만 원이라면, 3시간 절약으로 6만 원 이상의 가치가 있습니다.

**월 $20(약 3만 원)으로 매주 수십 시간을 절약**할 수 있다면, 이건 비용이 아니라 투자입니다.

## 정리

- Claude Code 비용은 **토큰(입력 + 출력)** 기반이고, 입력(특히 파일 읽기)이 대부분을 차지합니다

- `/cost`로 현재 사용량을 확인할 수 있습니다

- 비용 절약 핵심: **/compact 자주, 새 대화 적절히, CLAUDE.md 간결하게, Sonnet 기본, 파일 읽기 최소화**

- 처음에는 **Pro 플랜($20/월)**으로 시작하는 걸 추천합니다

- 모델은 기본 Sonnet, 복잡한 작업만 Opus로 전환하세요
