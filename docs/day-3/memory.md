# 기억력 키워주기 (메모리)

## 오늘의 목표

> 세션이 끝나도 기억이 유지되는 메모리 시스템을 활용하는 법을 배웁니다.

앞에서 컨텍스트(단기 기억)를 배웠습니다. 그런데 한 가지 문제가 있습니다. **대화를 끝내면 컨텍스트가 전부 사라집니다.** 내일 다시 Claude Code를 열면 어제 뭘 했는지 모릅니다.

이 문제를 해결하는 게 **메모리** 시스템입니다.

---

## 컨텍스트 vs 메모리

|  | 컨텍스트 (단기 기억) | 메모리 (장기 기억) |
| --- | --- | --- |
| 비유 | 칠판에 쓴 메모 | 노트에 적어둔 기록 |
| 유지 기간 | 현재 대화 동안만 | 대화가 끝나도 유지 |
| 저장 방식 | 자동 (대화하면 쌓임) | 파일에 저장 |
| 한계 | 모델에 따라 200K~1M 토큰 | 파일 크기 제한 없음 |
| 리셋 | 새 대화 시작하면 초기화 | 파일을 지우기 전까지 유지 |

컨텍스트가 **칠판**이라면, 메모리는 **노트**입니다. 칠판은 수업이 끝나면 지워지지만, 노트에 적어둔 건 다음에 다시 펼쳐볼 수 있습니다.

---

## Auto Memory: Claude가 알아서 기억하기

Claude Code는 대화 중에 중요하다고 판단한 내용을 **자동으로 기억**합니다. 이걸 Auto Memory라고 합니다.

예를 들어 대화 중에 이런 말을 하면:

`"이 프로젝트에서 변수명은 항상 camelCase로 써줘"`
Claude가 이 규칙을 기억하고, 다음 세션에서도 적용합니다. 여러분이 따로 설정할 필요 없이 자동으로 동작합니다.

> ℹ️ **정보**
>
> Auto Memory는 편리하지만, 무엇이 기억되었는지 직접 확인하기 어렵습니다. 확실하게 기억시키고 싶은 것은 다음에 나오는 CLAUDE.md 방식을 쓰는 게 좋습니다.

---

## CLAUDE.md 파일로 직접 관리하기

가장 확실한 메모리 관리 방법입니다. 프로젝트 루트에 `CLAUDE.md` 파일을 만들어두면, **매번 대화를 시작할 때 Claude가 자동으로 읽습니다.**

### 기본 사용법

`# 프로젝트 루트에서
claude "CLAUDE.md에 '변수명은 camelCase, 함수명은 동사로 시작' 규칙을 추가해줘"`
또는 직접 파일을 열어서 편집해도 됩니다:

`# CLAUDE.md
 
## 프로젝트 규칙
- 변수명은 camelCase로 작성
- 함수명은 동사로 시작 (getUser, createPost)
- 컴포넌트 파일은 PascalCase (UserProfile.tsx)
 
## 기술 스택
- Next.js 14 + TypeScript
- Tailwind CSS
- Prisma + PostgreSQL
 
## 주의사항
- .env 파일은 절대 커밋하지 않기
- API 키를 코드에 직접 넣지 않기`
이렇게 적어두면 다음에 Claude Code를 열었을 때 이 내용을 자동으로 인식합니다.

---

## 프로젝트 메모리 vs 글로벌 메모리

메모리는 **두 가지 범위**가 있습니다.

### 프로젝트 메모리 (CLAUDE.md)

프로젝트 폴더 안에 있는 `CLAUDE.md` 파일입니다. **해당 프로젝트에서만** 적용됩니다.

`~/my-project/
  ├── CLAUDE.md          ← 이 프로젝트에서만 적용
  ├── src/
  └── package.json`
프로젝트 A의 CLAUDE.md와 프로젝트 B의 CLAUDE.md는 완전히 별개입니다.

### 글로벌 메모리 (~/.claude/CLAUDE.md)

홈 디렉토리의 `.claude` 폴더에 있는 CLAUDE.md입니다. **모든 프로젝트에서** 적용됩니다.

`~/.claude/
  └── CLAUDE.md          ← 모든 프로젝트에서 적용`
|  | 프로젝트 메모리 | 글로벌 메모리 |
| --- | --- | --- |
| 위치 | `프로젝트/CLAUDE.md` | `~/.claude/CLAUDE.md` |
| 적용 범위 | 해당 프로젝트만 | 모든 프로젝트 |
| 넣을 내용 | 프로젝트별 규칙, 기술 스택 | 개인 코딩 스타일, 공통 선호사항 |
| 예시 | ”이 프로젝트는 Tailwind 사용" | "항상 한국어로 답변해줘” |

> ℹ️ **정보**
>
> 글로벌 메모리에는 “한국어로 답변”, “커밋 메시지는 한국어로” 같은 **개인 취향**을 넣으세요. 프로젝트 메모리에는 **프로젝트 고유 규칙**을 넣으세요.

---

## 실습: 메모리가 동작하는지 확인하기

직접 해봅시다.

**Step 1**: 작업 중인 프로젝트에서 CLAUDE.md에 규칙 하나를 추가합니다.

`claude "CLAUDE.md에 '코드 주석은 항상 한국어로 작성' 규칙을 추가해줘"`
**Step 2**: 대화를 완전히 종료합니다 (`/exit` 또는 `Ctrl+C`).

**Step 3**: Claude Code를 다시 실행하고, 코드를 작성하게 합니다.

`claude
> "간단한 계산기 함수를 만들어줘"`
**Step 4**: 생성된 코드의 주석이 한국어인지 확인합니다.

한국어로 주석이 달려 있다면, 메모리가 정상 작동하는 겁니다.

---

## 메모리에 넣으면 좋은 것 vs 안 넣어도 되는 것

### 넣으면 좋은 것

- **코딩 규칙**: 네이밍 컨벤션, 파일 구조 규칙

- **기술 스택**: 프레임워크, 라이브러리 버전

- **금지 사항**: “eval() 사용 금지”, “any 타입 사용 금지”

- **선호 사항**: “함수형 컴포넌트 우선”, “에러 메시지는 한국어로”

- **프로젝트 구조**: 주요 폴더의 역할

### 안 넣어도 되는 것

- **일시적 작업 지시**: “오늘은 로그인 기능만 작업” (이건 대화에서 말하면 됩니다)

- **일반 상식**: “변수는 의미 있는 이름으로” (Claude가 기본적으로 잘 합니다)

- **너무 자세한 코드 예시**: CLAUDE.md가 길어지면 오히려 컨텍스트를 낭비합니다

> ⚠️ **주의**
>
> CLAUDE.md는 간결하게 유지하세요. 이 파일도 매 대화마다 컨텍스트에 로딩됩니다. 너무 길면 정작 대화에 쓸 컨텍스트가 줄어듭니다. **A4 1페이지 이내**를 추천합니다.

---

## 정리

- **컨텍스트**(단기 기억)는 대화가 끝나면 사라지고, **메모리**(장기 기억)는 유지됩니다

- Claude Code는 중요한 내용을 **자동으로 기억**(Auto Memory)합니다

- 확실한 기억은 **CLAUDE.md 파일**에 직접 적어두세요

- **프로젝트 메모리**(프로젝트별)와 **글로벌 메모리**(전체 공통)를 구분해서 사용합니다

- CLAUDE.md는 간결하게, A4 1페이지 이내로 유지하세요
