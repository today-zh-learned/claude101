# 프롬프트 잘 쓰는 법

## 오늘의 목표

> Claude에게 더 정확하게 시키는 법을 배웁니다.

Day 1~2에서 Claude Code를 설치하고, 기본 사용법을 익혔습니다. 이제부터가 진짜입니다. **같은 도구를 써도 결과가 달라지는 이유**, 바로 프롬프트 때문입니다.

---

## 프롬프트가 뭔가요?

프롬프트는 **Claude에게 하는 말** 그 자체입니다. 특별한 문법이 아닙니다.

“파일 만들어줘”도 프롬프트이고, “auth.py의 login 함수에서 비밀번호 해싱 로직을 bcrypt로 바꿔줘”도 프롬프트입니다. 차이는 **얼마나 구체적인지**뿐입니다.

회사에서 신입에게 일을 시킬 때를 생각해보세요. “그거 좀 해놔”라고 하면 엉뚱한 결과가 나옵니다. “내일 오전 회의 자료 3페이지로 정리해서 공유 폴더에 넣어줘”라고 하면 원하는 게 나옵니다. Claude도 똑같습니다.

---

## 나쁜 프롬프트 vs 좋은 프롬프트

바로 비교해봅시다. 5가지 실제 상황입니다.

| # | 나쁜 프롬프트 | 좋은 프롬프트 |
| --- | --- | --- |
| 1 | ”코드 고쳐줘" | "auth.py의 login 함수에서 비밀번호 검증 로직을 수정해줘. 현재 평문 비교인데 bcrypt 해싱으로 바꿔야 해” |
| 2 | ”웹사이트 만들어줘" | "React로 포트폴리오 웹사이트를 만들어줘. 페이지는 소개, 프로젝트, 연락처 3개. 다크 테마로” |
| 3 | ”에러 고쳐줘" | "npm run build 하면 ‘Module not found: react-router-dom’ 에러가 나. 해결해줘” |
| 4 | ”테스트 만들어줘" | "utils/calculate.ts의 calculateTax 함수에 대해 Jest 단위 테스트를 만들어줘. 세율 0%, 10%, 경계값 케이스 포함” |
| 5 | ”정리 좀 해줘" | "src/components 폴더의 파일명을 전부 kebab-case로 바꿔줘. import 경로도 함께 수정해줘” |

패턴이 보이시나요? 좋은 프롬프트에는 항상 **무엇을**, **어디서**, **어떻게**가 들어있습니다.

---

## 5가지 핵심 기법

### 1. 구체적으로 말하기

가장 기본이면서 가장 중요합니다.

`❌ "README 만들어줘"
✅ "이 프로젝트의 README.md를 만들어줘. 프로젝트 소개, 설치 방법, 사용법, 라이선스 섹션을 포함하고, 설치 방법은 npm 기준으로 작성해줘"`
**실습**: 지금 작업 중인 폴더에서 “이 프로젝트 설명해줘”와 “이 프로젝트의 폴더 구조를 트리 형태로 보여주고, 각 폴더의 역할을 한 줄씩 설명해줘”를 각각 입력해보세요. 결과의 차이를 직접 느껴보시기 바랍니다.

### 2. 맥락 알려주기

Claude는 여러분의 상황을 모릅니다. 알려줘야 합니다.

`❌ "로그인 기능 만들어줘"
✅ "Express.js + MongoDB 프로젝트야. JWT 기반 로그인 기능을 만들어줘. 기존 User 모델은 models/user.js에 있어"`
기술 스택, 프로젝트 구조, 이미 있는 파일을 알려주면 Claude가 기존 코드와 어울리는 결과를 만들어줍니다.

### 3. 예시 보여주기

원하는 형태가 있으면 예시를 직접 보여주세요.

`"API 응답을 이 형식으로 만들어줘:
{
  "status": "success",
  "data": { ... },
  "message": "조회 완료"
}"`
Claude는 패턴 인식이 뛰어납니다. 예시 하나만 보여주면 나머지를 맞춰서 만들어줍니다.

### 4. 단계 나눠서 시키기

한 번에 너무 많이 시키면 결과가 흐려집니다.

`❌ "쇼핑몰 전체 백엔드 만들어줘"
✅ "먼저 상품 목록 API를 만들자. GET /api/products로 전체 상품을 조회하는 엔드포인트를 만들어줘"`
큰 작업은 작은 단계로 쪼개세요. 한 단계가 끝나면 확인하고 다음으로 넘어갑니다.

### 5. 결과 형식 지정하기

`❌ "비교해줘"
✅ "React vs Vue를 비교해줘. 표 형식으로 학습 곡선, 생태계, 성능, 채용 시장 기준으로 정리해줘"`
표, 목록, 코드 블록, 파일 형식 등 원하는 출력 형태를 지정하면 깔끔한 결과를 받을 수 있습니다.

## Plan 모드 활용하기

복잡한 작업이라면 바로 실행시키지 마세요. **계획부터 세우게 하는 게 낫습니다.**

`# shift+tab으로 Plan 모드 전환 후
"이 프로젝트에 다크모드를 추가하려고 해. 어떤 파일을 수정해야 하는지 계획을 세워줘"`
Claude가 수정할 파일 목록과 순서를 먼저 정리해줍니다. 계획을 확인한 뒤 “좋아, 진행해줘”라고 하면 됩니다.

> ℹ️ **정보**
>
> Plan 모드는 `shift+tab`으로 전환합니다. 복잡한 작업일수록 Plan 먼저, 실행은 나중에. 이 습관 하나가 실수를 크게 줄여줍니다.

## AI싱크클럽 노하우: 실전 프롬프트 예시

AI싱크클럽에서 23개 에이전트를 운영하면서 쌓은 팁입니다.

**에이전트에게 시키는 실제 프롬프트:**

`"AI싱크클럽 채널의 이번 주 영상 소재를 찾아줘.
- 최근 1주일 AI 도구 트렌드 기반
- 시리즈 B(활용편)에 맞는 소재
- 검색량과 경쟁도를 함께 분석해줘"`
이 한 문장에 **채널**(AI싱크클럽), **범위**(이번 주), **기준**(트렌드, 시리즈, 검색량)이 모두 들어있습니다. 구체적이니까 결과도 정확합니다.

> ⚠️ **주의**
>
> 프롬프트를 잘 쓰는 건 하루 만에 되지 않습니다. 하지만 “구체적으로 말하기” 하나만 의식해도 결과가 달라집니다. 오늘부터 Claude에게 말할 때 “무엇을, 어디서, 어떻게”를 넣어보세요.

## 정리

- 프롬프트는 특별한 문법이 아니라, Claude에게 하는 말입니다

- **구체적으로, 맥락과 함께, 예시를 보여주며, 단계를 나눠서, 형식을 지정해서** 말하면 됩니다

- 복잡한 작업은 Plan 모드로 계획을 먼저 세우세요

- 한 번에 완벽할 필요 없습니다. 결과를 보고 다시 시키면 됩니다
